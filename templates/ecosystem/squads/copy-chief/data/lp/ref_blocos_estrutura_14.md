# Estrutura dos 14 Blocos

Referência core com fórmulas e estruturas de cada bloco.

---

## BLOCO 1: HEADLINE

### Fórmula Padrão

```
Descubra como {PROMESSA PRINCIPAL} em {TEMPO ESPECÍFICO}, 
mesmo que {OBJEÇÃO/DIFICULDADE}.
```

### Variação Antes/Depois

```
{Headline padrão}

Veja o antes e depois de pessoas que {SITUAÇÃO INICIAL NEGATIVA} 
e que, após {NOME DO PRODUTO}, conseguiram {RESULTADO POSITIVO}.
```

### Checklist

| Elemento | Verificação |
|----------|-------------|
| Promessa | Específica e mensurável |
| Tempo | Crível para o resultado |
| Objeção | Principal do avatar (Fase 3) |

---

## BLOCO 2: DEPOIMENTOS

### Estrutura

```
[RESUMO DE DESTAQUE - 1 linha]

"{Quote curta ou resultado específico}"
— Nome (ou inicial) + contexto opcional
```

### 4 Ângulos Obrigatórios

| Ângulo | Foco |
|--------|------|
| Resultado | Transformação alcançada |
| Facilidade | Simplicidade de uso |
| Rapidez | Velocidade do resultado |
| Ceticismo | Dúvida inicial vencida |

### Checklist

| Elemento | Verificação |
|----------|-------------|
| Quantidade | Mínimo 4 depoimentos |
| Resumos | Escaneáveis (1 linha) |
| Variedade | 4 ângulos diferentes |

## BLOCO 3: RUMINAÇÃO

### Estrutura em 4 Partes

```
PARTE 1 - Agitação:
"Você não aguenta mais {AÇÃO REPETITIVA/DOR COTIDIANA} 
e ver sempre {RESULTADO FRUSTRANTE}."

PARTE 2 - Validação:
"Eu sei. Você não aguenta mais {DOR ESPECÍFICA}. Estou certo?"

PARTE 3 - Ruminações (3-4 pensamentos internos):
- "{Pensamento sobre a dor principal}"
- "{Pensamento sobre frustração}"
- "{Pensamento sobre tentativas falhas}"
- "{Pensamento sobre desejo reprimido}"

PARTE 4 - Transição:
"Agora eu te pergunto... Se você pudesse ter {RESULTADO/BENEFÍCIO}, 
sem {OBJEÇÃO MAIOR}, isso mudaria sua vida?
Se a sua resposta for SIM, o {NOME DO PRODUTO} é para você."
```

### Checklist

| Elemento | Verificação |
|----------|-------------|
| Linguagem | VOC exato quando disponível |
| DRE | Presente nas ruminações |
| Transição | Conecta dor com solução |

## BLOCO 4: PASSO A PASSO

### Estrutura

```
Esses são os 3 passos para {RESULTADO PRINCIPAL}

Passo 1: {NOME DO PASSO}
{Breve descrição - 1-2 linhas}

Passo 2: {NOME DO PASSO}
{Breve descrição - 1-2 linhas}

Passo 3: {NOME DO PASSO}
{Breve descrição - 1-2 linhas}
```

### Progressão Lógica

| Passo | Função |
|-------|--------|
| 1 | Entender/Descobrir |
| 2 | Aplicar/Executar |
| 3 | Colher/Alcançar |

### Checklist

| Elemento | Verificação |
|----------|-------------|
| Derivação | Baseado no MUS (Fase 6) |
| Simplicidade | Cada passo acionável |
| Progressão | Lógica clara |

## BLOCO EXTRA: BENEFÍCIOS

### Estrutura

```
Com esse passo a passo você vai:

- {Benefício 1 - resultado}
- {Benefício 2 - resultado}
- {Benefício 3 - resultado}
- {Benefício 4 - resultado}
```

### Checklist

| Elemento | Verificação |
|----------|-------------|
| Foco | RESULTADOS, não features |
| Verbos | Ação ou estado desejado |
| Conexão | Desejos do avatar (Fase 3) |

## BLOCO 5: ENTREGÁVEIS

### Estrutura por Entregável

```
{NOME DO ENTREGÁVEL}
{Descrição 1-2 linhas: O que é + Benefício + Resultado}
```

### Tipos Comuns

| Tipo | Estrutura do Nome |
|------|-------------------|
| Checklist | Checklist de {Promessa Principal} |
| Template | Template de {Aplicação Prática} |
| Guia | Guia de {Ações} |
| Lista | Lista de {Erros a Evitar} |

### Checklist

| Elemento | Verificação |
|----------|-------------|
| Nomes | Próprios, não genéricos |
| Descrição | Foco em RESULTADO |
| Contexto | Tempo/objeção quando relevante |
