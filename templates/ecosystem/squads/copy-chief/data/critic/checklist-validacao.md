# Checklist de Validação
## 16 Pontos para Copy de Alta Conversão

> **Usar para:** Verificação sistemática antes de aprovar qualquer copy.
> **Contexto:** Combina 6-Question Hook Test + RMBC Validation + Body Consistency Check.

---

## Bloco 1: 6-Question Hook Test (6 pontos)

Cada headline/lead deve passar em pelo menos 4 de 6:

### Q1. Oferece RECOMPENSA por ler?
- [ ] Benefício claro está presente no hook
- [ ] Avatar sabe o que ganha se continuar

**Teste:** "O que o avatar ganha lendo isso?"

### Q2. Tem ESPECÍFICOS que intrigam?
- [ ] Números concretos presentes
- [ ] Detalhes que criam curiosidade
- [ ] Prazo ou quantidade definidos

**Teste:** "Tem algo que posso visualizar?"

### Q3. Dispara EMOÇÃO FORTE existente?
- [ ] Conecta com DRE identificada
- [ ] Não é emoção inventada
- [ ] Intensidade adequada ao nicho

**Teste:** "O avatar já sente isso antes de ler?"

### Q4. Tem elemento de INTRIGA?
- [ ] Cria gap de curiosidade
- [ ] Promete revelar algo
- [ ] Desafia crença comum

**Teste:** "O avatar PRECISA saber o que vem depois?"

### Q5. É CLARA e simples?
- [ ] Entendida em <5 segundos
- [ ] Sem jargão desnecessário
- [ ] Uma ideia principal (não várias)

**Teste:** "Minha avó entenderia?"

### Q6. Conecta com EVENTOS ATUAIS?
- [ ] Relevância temporal
- [ ] Contexto reconhecível
- [ ] Não parece datada

**Teste:** "Faz sentido para o avatar HOJE?"

**Resultado Hook:** __/6 (mínimo 4 para PASS)

---

## Bloco 2: RMBC Validation (5 pontos)

Para MUP/MUS/Mecanismo:

### R - Reason Why (Por que funciona)
- [ ] Explica CAUSA RAIZ do problema
- [ ] Conexão causa → solução é lógica
- [ ] Avatar consegue re-explicar para outro

**Score:** __/1

### M - Mechanism (Mecanismo Único)
- [ ] Tem Gimmick Name próprio
- [ ] Tem Origin Story
- [ ] Tem Ingrediente/Componente Hero
- [ ] Pode ser visualizado mentalmente

**Score:** __/1 (todos sub-itens)

### B - Believability (Credibilidade)
- [ ] Passaria para amigo cético
- [ ] Tem provas/evidências
- [ ] Sem claims impossíveis
- [ ] Ressalvas realistas presentes

**Score:** __/1 (todos sub-itens)

### C - Connection (Conexão)
- [ ] Usa linguagem VOC (frases exatas)
- [ ] Toca DRE do avatar
- [ ] Gera identificação ("isso sou eu")
- [ ] Esperança realista (não fantasia)

**Score:** __/1 (todos sub-itens)

### Espelhamento MUP-MUS
- [ ] MUS é solução DIRETA para MUP
- [ ] Não há gap lógico entre problema e solução

**Score:** __/1

**Resultado RMBC:** __/5 (mínimo 4 para PASS)

---

## Bloco 3: Body Consistency Check (5 pontos)

Para copy completa (VSL, LP, criativo):

### C1. Consistência de Voz
- [ ] Tom mantido do início ao fim
- [ ] Sem mudanças bruscas de estilo
- [ ] Nível de formalidade consistente

**Score:** __/1

### C2. Progressão Emocional
- [ ] Arco claro: Dor → Esperança → Solução → Urgência
- [ ] Não pula etapas
- [ ] Intensidade cresce progressivamente

**Score:** __/1

### C3. Prova Social Adequada
- [ ] Depoimentos específicos (não genéricos)
- [ ] Quantidade apropriada ao tipo
- [ ] Credibilidade específica (nome, idade, cidade, resultado)

**Score:** __/1

### C4. Call-to-Action Claro
- [ ] Ação específica solicitada
- [ ] Urgência/escassez legítima
- [ ] Sem ambiguidade do próximo passo

**Score:** __/1

### C5. Objeções Endereçadas
- [ ] Preço justificado (valor > custo)
- [ ] Garantia presente
- [ ] "E se não funcionar?" respondido

**Score:** __/1

**Resultado Body:** __/5 (mínimo 4 para PASS)
