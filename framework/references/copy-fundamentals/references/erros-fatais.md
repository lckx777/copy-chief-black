# Erros Fatais - O Que NUNCA Fazer

> **Memorizar:** Estes erros destroem conversao mesmo em copy "tecnicamente correta".

---

## Os 7 Erros Tecnicos (Victor Mound)

### Erro 1: Desorganizacao do Documento

**Problema:** Copywriter imagina anuncio completo, mas documento para editor nao contem todas informacoes.

**Omissoes frequentes:**
- Link do avatar/ator
- Referencia de edicao (link de anuncio similar)
- Musica desejada
- Comentarios explicativos sobre intencao
- Referencia visual para cenas especificas
- Tom de voz desejado

**Consequencia:** Editor interpreta a sua maneira. Resultado final diferente do imaginado. Anuncio nao performa.

---

### Erro 2: Prolixidade

**Problema:** Explicar em 3 paragrafos o que caberia em 2 frases.

**Teste:** "Eu bocejaria lendo isso?" Se sim, esta prolixo.

**Solucao:** Para cada paragrafo, pergunte: "Posso falar isso de forma mais curta?"

---

### Erro 3: Seguir Estrutura Cegamente

**Problema:** Aplicar estrutura (hook → historia → mecanismo → prova → CTA) roboticamente, sem avaliar se esta interessante.

**Sintoma:** Anuncio "esta certo" tecnicamente, mas e chato. Tem todos elementos, mas nao prende.

**Solucao:** Estrutura e guia, nao prisao. Releia como CONSUMIDOR: "Eu pararia para ver isso?"

---

### Erro 4: Trocar Elementos Essenciais

**Conceito de Cicero:** Todo anuncio tem ESSENCIAIS e ACIDENTAIS.

| ESSENCIAIS (NAO MEXER) | ACIDENTAIS (PODE TESTAR) |
|------------------------|--------------------------|
| Beneficio principal | Ordem das informacoes |
| Future pacing | Palavras exatas |
| Pontos de dor | Detalhes da historia |
| Mecanismo diferenciado | Elementos visuais secundarios |
| Prova | |

**Erro do junior:** Quer "ser criativo" e troca ESSENCIAIS por ideias aleatorias. Remove future pacing porque "achou desnecessario". Tira dor porque "ficou negativo demais".

---

### Erro 5: Falta de Senso Critico na Revisao

**Problema:** Copywriter nao rele o proprio anuncio com olhar critico.

**Perguntas obrigatorias:**
- Eu pararia para ver isso no feed?
- Isso esta chato?
- Essa frase e necessaria?
- Estou sendo prolixo aqui?
- O hook esta forte o suficiente?
- Existe alguma parte que eu pularia?

**Solucao:** Apos escrever, espere 30 minutos. Volte e leia como lead cetico, apressado, desconfiado.

---

### Erro 6: Hook Sem os 3 Elementos

**Problema:** Hook que nao qualifica, nao emociona ou nao gera curiosidade.

**Exemplo ruim:**
> "Voces nao vao acreditar no que aconteceu!"
- Nao qualifica (quem e o publico?)
- Nao gera emocao especifica
- Curiosidade generica

**Solucao:** QUALIFICACAO + EMOCAO + CURIOSIDADE

---

### Erro 7: Usar "Isso Aqui" no Lugar de Gimmick Name

**Problema:**
```
"Eu usei ISSO AQUI para emagrecer."
"Faca ISSO todo dia."
"Descobri ISSO e mudou minha vida."
```

**Por que nao funciona:**
- Nao qualifica (isso o que?)
- Nao gera curiosidade especifica
- Parece clickbait generico

**Solucao:**
```
"Eu usei o TRUQUE DO GENGIBRE CONGELADO para emagrecer."
"Faca o RITUAL DO VINAGRE todo dia."
"Descobri o METODO JAPONES e mudou minha vida."
```

**Regra:** Se existe gimmick name para o mecanismo, ele DEVE estar no hook.

## Conceito de Cicero - Expandido

### Elementos ESSENCIAIS (Validados pelo Mercado)

1. **Beneficio Principal** - A promessa core
2. **Future Pacing** - Projecao do resultado
3. **Pontos de Dor** - Identificacao com sofrimento
4. **Mecanismo Diferenciado** - O "como" unico
5. **Prova** - Evidencia de que funciona

**NUNCA remover ou substituir por "ideias criativas".**

### Elementos ACIDENTAIS (Testaveis)

- Ordem especifica das informacoes
- Palavras exatas usadas
- Detalhes da historia
- Elementos visuais secundarios

Se tirar ou mudar, anuncio continua funcionando.

## Anti-Patterns Comuns

| Anti-Pattern | Problema | Correcao |
|--------------|----------|----------|
| Numeros redondos | Parecem inventados | "12,4kg" nao "12kg" |
| "Estudos mostram" | Sem especificar fonte | "Harvard 2023 publicou" |
| Formato saturado | Lead ja sabe que e venda | Buscar novo formato |
| Remover frases de poder | Perde forca do hook | Manter ou substituir equivalente |
| Copy "PowerPoint" | Tom de consultoria | Reescrever como conversa |
| Elogiar demais | Lead desconfia | Manter tom realista |

## Erro 8: Ausencia de Voz

**O problema:**
Copy tecnicamente correta mas sem PERSONALIDADE. Parece documento corporativo, nao conversa real.

**Sintomas:**
- Frases longas e formais
- Linguagem neutra, sem emocao
- Leitura nao engaja (correto mas chato)
- Parece "feita por IA"

**Exemplo:**

SEM VOZ:
> "Este metodo foi desenvolvido para auxiliar pessoas que desejam melhorar seu desempenho em provas atraves de tecnicas cientificamente comprovadas."

COM VOZ:
> "Voce estuda. Estuda. Estuda. E na hora da prova? Erra o que sabia."

**Solucao:**
Ler em voz alta. Se soa como apresentacao de PowerPoint, reescrever como desabafo no bar.

## Erro 9: Estilo Hardcoded (Tom Fixo) - CRÍTICO v5.0

**O problema:**
Aplicar o MESMO tom/emoção em TODOS os criativos, independente do nicho, DRE ou briefing.

**Exemplos do erro:**
- Usar "raiva" em criativo de maternidade (tom inadequado)
- Usar "deboche" em criativo de luto/saúde grave
- Usar gírias de um nicho em outro
- **NOVO v5:** Assumir que "BLACK = raiva sempre"

**Por que acontece:**
Copywriter aprende que "raiva vende" em um nicho (ex: concursos) e replica universalmente.
**v5:** Interpretam "BLACK" como tom fixo, não como intensidade emocional.

**Solução v5:**
1. Identificar DRE do público (não assumir medo/raiva)
2. Ler 2-3 swipes do nicho específico
3. Extrair TOM dominante (vulnerabilidade ↔ raiva spectrum)
4. CALIBRAR tom baseado em: DRE + nicho + swipes
5. BLACK = intensidade (níveis 4-5), NÃO tom específico

**Exemplos de Tom Calibrado:**

| Nicho | DRE | Tom Calibrado |
|-------|-----|---------------|
| Concursos | Frustração/Raiva | Raiva contra sistema |
| Maternidade | Culpa/Medo | Vulnerabilidade + esperança |
| Emagrecimento | Vergonha | Vulnerabilidade + descoberta |
| Riqueza | Desejo/Raiva | Raiva contra elite ou ambição |

**Regra v5:** Tom é CALIBRADO (não fixo). Descobrir via briefing + swipes, não hardcodar.
**BLACK = intensidade 4-5, não tom específico.**

## Erro 10: Ignorar Swipes

**O problema:**
Inventar estilo do zero ao inves de modelar dos swipes validados.

**Sintomas:**
- Girias inventadas que nao existem no nicho
- Tom desconectado do que escala no mercado
- Copy "original" que nao converte
- Linguagem artificial

**Exemplo:**

IGNOROU SWIPES:
> Criou giria "Metodo Explosao Mental" para concursos (nao existe no nicho)

MODELOU SWIPES:
> Usou "tomando fumo", "famosinhas", "hack" (girias reais do nicho)

**Solucao:**
ANTES de escrever qualquer palavra:
1. Identificar swipes do nicho
2. Ler 2-3 swipes
3. Documentar tom/girias/ritmo
4. MODELAR estilo, nao inventar

**Regra:** Swipes sao OBRIGATORIOS. Nao existe criativo sem consulta.
