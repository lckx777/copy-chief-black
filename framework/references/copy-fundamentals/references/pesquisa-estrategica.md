# Pesquisa Estratégica - VOC Discovery e Triangulação

> **Aplicar em:** TODA pesquisa de copy.
> Pesquisa bem feita = copy que converte.
> Fundamentação: Metodologia Vitor Mound

---

## Por Que Pesquisa Importa

A pesquisa permite:
- Encontrar pontos de conexão com a realidade
- Usar celebridades/autoridades com credibilidade
- Descobrir contextos exploráveis
- Criar Big Ideas baseadas em informação real
- Aumentar especificidade com dados reais

---

## Fontes para Pesquisar

### Fontes de VOC (Voice of Customer)

| Fonte | O que buscar |
|-------|-------------|
| **Conteúdo orgânico viral** (Instagram/TikTok) | Linguagem, formatos que viralizam |
| **Comentários em posts virais** | Linguagem REAL do público |
| **Grupos de Facebook** | Dores, desejos, objeções |
| **Reviews de produtos** | O que gostam, o que reclamam |
| **Reclame Aqui** | Frustrações, palavras de raiva |
| **Fóruns especializados** | Terminologia técnica do nicho |
| **Subreddits** | Discussões profundas, dores ocultas |
| **YouTube (comentários)** | Reações a conteúdo do nicho |

## O que Buscar

### 1. Linguagem Real

Como o público REALMENTE fala sobre o problema?
Não como você imagina — como ELES falam.

```
❌ "Sinto-me frustrada com minha condição de saúde"
✅ "Cansei dessa p**** de não conseguir emagrecer"
```

### 2. Dores Não Endereçadas

Quais problemas o mercado NÃO está resolvendo?
Onde estão as lacunas?

### 3. Desejos Não Atendidos

O que o público QUER que ninguém está oferecendo?

### 4. Objeções Recorrentes

Quais são as desculpas/razões para não comprar?
Quais medos impedem a ação?

### 5. Histórias Reais de Transformação

Depoimentos autênticos com detalhes específicos.
Números, datas, situações concretas.

## Priorização por Intensidade

Nem toda dor/desejo/objeção tem o mesmo peso.

**Hierarquia de prioridade:**

```
1. DORES (maior peso) → O que MACHUCA o avatar
2. DESEJOS (peso médio) → O que ele QUER
3. OBJEÇÕES (menor peso) → Por que ele NÃO age
```

**Dentro de cada categoria:**

Priorizar por INTENSIDADE emocional:
- Intensidade 5/5 → Usar no hook
- Intensidade 4/5 → Usar no corpo
- Intensidade 3/5 → Usar como reforço
- Intensidade 1-2/5 → Descartar

---

## Triangulação

**Regra:** Quote encontrada em múltiplas fontes = maior peso.

Se a mesma dor aparece em:
- Comentários do Instagram
- Reviews do Reclame Aqui
- Posts de Facebook

→ Essa dor é VALIDADA. Usar com prioridade.

**Documentar triangulação:**

| Quote | Fonte 1 | Fonte 2 | Fonte 3 | Peso |
|-------|---------|---------|---------|------|
| "Estudo 8h e reprovo" | YouTube | Reddit | TikTok | ALTO |
| "Não tenho tempo" | Instagram | - | - | MÉDIO |

## Mapeamento dos 4 Dominantes

A pesquisa deve alimentar os 4 Dominantes do avatar:

| Dominante | O que buscar |
|-----------|-------------|
| **EMOTION** | Qual emoção dominante? Raiva? Medo? Vergonha? |
| **DESIRE** | Qual o desejo mais intenso? |
| **OBJECTION** | Qual a maior objeção/barreira? |
| **PROOF** | Que tipo de prova eles precisam? |

---

## Pesquisa para Uso de Celebridades

Quando usar celebridade/autoridade no anúncio:

**Checklist de pesquisa:**
```
[ ] O que ela já disse publicamente sobre o tema?
[ ] Que produtos/métodos ela já endossou?
[ ] Qual o contexto da vida dela que conecta com o produto?
[ ] Existe alguma declaração que pode ser usada?
[ ] Qual a percepção pública atual sobre ela?
[ ] Existe alguma polêmica recente que pode afetar?
```

## Pesquisa Competitiva

### Biblioteca de Anúncios

**O que verificar:**
```
[ ] Quais anúncios estão rodando há mais tempo?
[ ] Quais foram pausados rapidamente?
[ ] Quais hooks estão sendo repetidos?
[ ] Quais Big Ideas estão saturadas?
[ ] Quais formatos estão dominando?
```

**Objetivo:** Identificar tanto o que funciona (para modelar) quanto o que está saturado (para evitar).

---

## Processo de Pesquisa

```
1. Definir nicho/oferta
2. Listar fontes relevantes (ver tabela acima)
3. Extrair quotes literais (mínimo 50 por fonte)
4. Classificar por tipo (dor/desejo/objeção)
5. Atribuir intensidade (1-5)
6. Verificar triangulação
7. Mapear nos 4 Dominantes
8. Documentar em arquivo de research
```
