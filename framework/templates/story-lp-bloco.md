## Acceptance Criteria (Especificos por Tipo de Bloco)

### Cross-Bloco (Todos)
- [ ] Especificidade Score >= 8
- [ ] Zero palavras banidas
- [ ] Linguagem da VOC (nao marketer-speak)
- [ ] Transicao de entrada fluida (conecta com bloco anterior)
- [ ] Transicao de saida cria ponte (prepara proximo bloco)
- [ ] Escaneabilidade: bullets, espacamento, nao e "muro de texto"

### Bloco 01 (Header + Headline)
- [ ] Headline entrega Big Promise em <= 10 palavras
- [ ] Sub-headline qualifica ou cria curiosidade
- [ ] Headline passaria no "Bar Test" (diria numa conversa?)
- [ ] Headline contem elemento do One Belief

### Bloco 02 (Video/Lead)
- [ ] Se VSL: thumbnail estrategica descrita
- [ ] Se TSL: lead copy segue NUUPPECC selecionado
- [ ] CTA visivel abaixo do video/lead

### Bloco 03 (Problema Agitado)
- [ ] Sinestesia emocional presente (detalhes sensoriais)
- [ ] Minimo 3 sintomas/dores listados (bullets)
- [ ] DRE ativada nivel >= 3
- [ ] Leitor se identifica ("isso sou eu")

### Bloco 04 (Vilao Revelado)
- [ ] Vilao e EXTERNO (tira culpa do avatar)
- [ ] Vilao e especifico (nao generico)
- [ ] Cria indignacao ("me enganaram")
- [ ] Conecta com por que solucoes anteriores falharam

### Bloco 05 (Mecanismo/Descoberta)
- [ ] MUP nomeado (Sexy Cause aparece)
- [ ] Reframe do problema ("nao era o que voce pensava")
- [ ] Cria "aha moment" (mudanca de paradigma)
- [ ] Evidencia/prova do mecanismo

### Bloco 06 (Apresentacao Solucao)
- [ ] MUS nomeado (Gimmick Name aparece)
- [ ] Conecta diretamente com MUP ("se o problema e X, a solucao e Y")
- [ ] Produto apresentado como resposta LOGICA (nao forcada)
- [ ] "Para quem e" claramente definido

### Bloco 07 (Beneficios + Transformacao)
- [ ] Features traduzidas em Benefits
- [ ] Antes/Depois com especificidade
- [ ] Futuro desejado pintado com detalhes sensoriais
- [ ] Minimo 5 beneficios listados

### Bloco 08 (Prova Social)
- [ ] Depoimentos com nomes e detalhes especificos
- [ ] Numeros/resultados concretos
- [ ] Variedade de perfis (nao todos iguais)
- [ ] Contexto do resultado (quanto tempo, situacao inicial)

### Bloco 09 (Autoridade do Expert)
- [ ] Credenciais relevantes (nao genericas)
- [ ] Historia de origem conecta com o mecanismo
- [ ] Humanizado (nao so curriculum)

### Bloco 10 (Stack)
- [ ] Minimo 5 itens no stack
- [ ] Valor individual para cada item
- [ ] Valor total vs preco real (ancoragem)
- [ ] Cada item tem nome proprio + beneficio

### Bloco 11 (Garantia)
- [ ] Tipo de garantia explicito
- [ ] Prazo claro
- [ ] Tom confiante (nao defensivo)
- [ ] Inversao de risco evidente

### Bloco 12 (Preco + CTA)
- [ ] Ancora de preco visivel
- [ ] Economia calculada
- [ ] CTA com acao clara ("Quero [resultado]")
- [ ] Formas de pagamento mencionadas
- [ ] Selos de seguranca

### Bloco 13 (FAQ)
- [ ] Top 5 objecoes respondidas
- [ ] Respostas curtas e diretas
- [ ] Cada resposta reforca um beneficio
- [ ] Tom empatetico (nao defensivo)

### Bloco 14 (CTA Final + Urgencia)
- [ ] Resumo da transformacao (1-2 frases)
- [ ] Urgencia REAL (nao falsa)
- [ ] Consequencia de nao agir (DRE ativada)
- [ ] CTA repetido com variacao
