## Section 1: Close Stack (7 Layers)

| Layer | Type | Present? | Executed Correctly? |
|-------|------|----------|---------------------|
| 1 | SUMMARY CLOSE -- itemized value stack | [ ] | [ ] Every item named + dollar value |
| 2 | REASON-WHY CLOSE -- why this deal exists | [ ] | [ ] Specific, slightly imperfect, external reason |
| 3 | RISK REVERSAL CLOSE -- named guarantee | [ ] | [ ] Specific outcome, measurable, time-bound |
| 4 | URGENCY CLOSE -- hard deadline | [ ] | [ ] Date + time + consequence + reason-why |
| 5 | SCARCITY CLOSE -- limited quantity | [ ] | [ ] Exact number + logistic reason |
| 6 | BONUS CLOSE -- bonuses exceeding price | [ ] | [ ] Each bonus named, valued, reason for inclusion |
| 7 | P.S. CLOSE -- 2-3 P.S. sections | [ ] | [ ] Each P.S. works as standalone selling argument |

**Minimum requirement:** All 7 layers present. Layers 1-5 are MANDATORY. Layers 6-7 are STRONGLY RECOMMENDED.

- [ ] Layers revealed PROGRESSIVELY (not dumped at once)
- [ ] If/Then Close present: "If you're serious about [desire], then..."
- [ ] Trial closes placed THROUGHOUT the copy, not just at the end
- [ ] Close architecture follows order: summary -> reason-why -> risk reversal -> urgency -> scarcity -> bonus -> P.S.

## Section 3: Urgency/Scarcity Audit

### Urgency Quality

- [ ] Every urgency claim has a REASON (not just "buy now")
- [ ] Reason is SPECIFIC and SLIGHTLY IMPERFECT (not too perfect)
- [ ] Reason is EXTERNAL to the sales process (third party, logistics, nature)
- [ ] Prospect should feel LUCKY, not pressured (right place, right time)
- [ ] Urgency escalates through the close (mild -> severe -> final warning)

### Scarcity Quality

- [ ] Scarcity has LOGISTIC reason (not just "limited spots")
- [ ] Exact numbers used ("347 remaining" not "limited")
- [ ] Scarcity is CONSEQUENCE of quality/service level
- [ ] Scarcity type is at least "production" level credibility
- [ ] No countdown timers that reset
- [ ] Scarcity connected to something the prospect VALUES

### Deadline Enforcement

- [ ] Deadline will be ENFORCED (not extended after it passes)
- [ ] Enforcement proof present: evidence of past enforcement
- [ ] Post-deadline plan exists (what happens to late requests)

## Section 4: CTA Stacking

| CTA Layer | Purpose | Present? |
|-----------|---------|----------|
| CTA #1 | WHAT to do | [ ] "Click the button below right now" |
| CTA #2 | HOW to do it | [ ] "Fill in name, select package, done" |
| CTA #3 | WHAT HAPPENS when you do | [ ] "Within 60 seconds, you'll receive..." |
| CTA #4 | CONSEQUENCE of NOT doing | [ ] "If you leave, this offer disappears" |
| CTA #5 | IF/THEN close | [ ] "If you're serious about [desire], then [action] is the only logical step" |

**Minimum requirement:** 3 CTAs present. CTA #5 (If/Then) STRONGLY recommended.

- [ ] Imperative verbs used: "Click", "Call", "Enter", "Select"
- [ ] Process described step-by-step (absurdly specific)
- [ ] Immediate aftermath painted (next 60 seconds)
- [ ] Button text is specific: "Send Me [Product] Now" not "Submit"
- [ ] Negative CTA present (what they LOSE)
- [ ] Trial closes placed THROUGHOUT the copy (not just at end)

## Section 5: Risk Reversal / Guarantee

| Criterion | Check |
|-----------|-------|
| Guarantee is NAMED? | [ ] (e.g., "The Iron-Clad Triple Lock Guarantee") |
| Tied to MEASURABLE outcome? | [ ] (e.g., "$500 in revenue in 90 days") |
| Specific TIMEFRAME? | [ ] (e.g., "180 days") |
| Has reason-why for generosity? | [ ] (e.g., "I can do this because 94.7% of...") |
| Placed next to CTA? | [ ] |
| At least Level 2 strength? | [ ] |
| Guarantee itself functions as a selling argument? | [ ] |
| Guarantee restated in P.S.? | [ ] |

**Guarantee Strength Levels:**
- Level 1: "Money back if not satisfied" (WEAK -- avoid)
- Level 2: "Full refund, no questions, keep bonuses" (MODERATE -- minimum)
- Level 3: "Double money back if [result] doesn't happen" (STRONG -- target)
- Level 4: "[Result] or I pay you $[amount]" (NUCLEAR)
- Level 5: "Better-than-money-back: refund + keep everything + compensation" (SUPREME)

### Value Stack

| Component | Present? | Named? | Dollar Value? | Reason for Inclusion? |
|-----------|----------|--------|---------------|----------------------|
| Core Product | [ ] | [ ] | [ ] | N/A |
| Bonus #1 (Accelerator) | [ ] | [ ] | [ ] | [ ] |
| Bonus #2 (Complementary Tool) | [ ] | [ ] | [ ] | [ ] |
| Bonus #3 (Templates/Swipes) | [ ] | [ ] | [ ] | [ ] |
| Bonus #4 (Access/Support) | [ ] | [ ] | [ ] | [ ] |
| Bonus #5 (Case Studies/Proof) | [ ] | [ ] | [ ] | [ ] |
| Fast Action Bonus | [ ] | [ ] | [ ] | [ ] |

**Minimum requirement:** Core + 4 bonuses (5 items total). Fast Action Bonus recommended.

- [ ] Total perceived value > 10x the actual price
- [ ] At least 1 bonus value EXCEEDS the core product price
- [ ] Each bonus solves a SECONDARY objection ("no time?" / "will it work for me?")
- [ ] Bonuses NOT available separately ("You cannot buy these individually")
- [ ] Value stack formatted visually (itemized list with dollar values, dotted lines or columns)
- [ ] "Just" used before actual price ("Your investment today: just $197")

### Price Presentation

- [ ] Price presented AFTER full value stack (never before)
- [ ] Compared to cost of the PROBLEM (annual/lifetime cost)
- [ ] Compared to cost of ALTERNATIVES (professional fees, competitor prices)
- [ ] Daily cost breakdown present ("less than $1.37/day")
- [ ] Payment plan offered alongside single-pay option
- [ ] Payment plan total is 10-20% more than single pay
- [ ] Zero apology for the price ("This isn't cheap. Here's why it shouldn't be.")

### Payment Plan

- [ ] Payment plan offered as accommodation, not featured option
- [ ] Payment amount compared to trivial daily expense
- [ ] Single-pay option positioned as the SMART choice (savings)
- [ ] Payment plan total and number of payments clearly stated

- [ ] Deadline will be ENFORCED -- no extensions, no exceptions

## Section 9: "No B.S." Voice Calibration

### Directness Check

- [ ] Zero hedge words: "might", "could", "possibly", "perhaps", "maybe"
- [ ] Zero corporate speak: "leverage", "synergy", "optimize", "empower", "journey"
- [ ] Zero filler: "In today's fast-paced world", "As we all know", "It goes without saying"
- [ ] Uses contractions: "don't" not "do not", "you'll" not "you will"
- [ ] Short sentences. Punchy paragraphs. 1-3 sentences per paragraph max.
- [ ] "Look," and "Here's the thing:" used as lead-ins
- [ ] "Let me be blunt:" before hard truths
- [ ] Reads like ONE PERSON talking to ANOTHER PERSON (not a committee memo)

### Polarization Check

- [ ] At least one BOLD opinion or controversial stance
- [ ] Position clearly stated and defended (not fence-sitting)
- [ ] Copy would make at least some people DISAGREE (that's the point)
- [ ] Zero wishy-washy language: "some people think" / "there are arguments on both sides"

### Damaging Admission Placement

- [ ] Exactly ONE strategic admission present
- [ ] Placed at 60-70% through the piece (AFTER proof, BEFORE close)
- [ ] Admission is REAL, not manufactured vulnerability
- [ ] Admission REFRAMED as filter: "If that bothers you, this isn't for you"
- [ ] Admission STRENGTHENS positioning: "It's expensive BECAUSE..."
- [ ] Does NOT undermine the core promise
- [ ] Does NOT appear after the CTA

### P.S. Architecture

- [ ] Minimum 2 P.S. sections present
- [ ] P.S. = strongest benefit or strongest proof (standalone selling argument)
- [ ] P.P.S. = hardest deadline or strongest consequence
- [ ] P.P.P.S. (optional) = guarantee restatement or new angle
- [ ] Each P.S. works as a COMPLETE mini-ad (reader may only read these)
- [ ] Each P.S. is 3-4 sentences max

### Emphasis System

- [ ] ALL CAPS used for core promise, deadline, critical warning (not entire paragraphs)
- [ ] Bold used for named elements, key numbers, product name
- [ ] Emphasis creates visual scanning paths (even skimmers get the message)
- [ ] Not over-used (when everything is emphasized, nothing is)

---

## Section 10: Takeaway Selling

- [ ] At least one DISQUALIFICATION present: "This isn't for everyone"
- [ ] Specific criteria that EXCLUDE certain people
- [ ] Prospect must feel they need to PROVE they qualify
- [ ] Removal threat present: "I almost didn't offer this" or "I reserve the right to refuse"
- [ ] Disqualification STRENGTHENS perceived value (not everyone can have it = valuable)
