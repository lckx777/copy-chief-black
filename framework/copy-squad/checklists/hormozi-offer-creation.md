## Section 1: Market Selection (Starving Crowd)

Before building ANY offer, validate the market against all four criteria.

| # | Criterion | Question | Score (1-10) | Notes |
|---|-----------|----------|--------------|-------|
| 1 | **Pain** | Do prospects have a DESPERATE need (not casual want)? | ___/10 | |
| 2 | **Purchasing Power** | Can they afford OR access the money to pay? | ___/10 | |
| 3 | **Easy to Target** | Can you reach them efficiently at scale? | ___/10 | |
| 4 | **Growing** | Is the market expanding (not shrinking)? | ___/10 | |

**Threshold:** Average score must be >= 7/10 across all four criteria.

- [ ] Pain is ACUTE, not chronic-and-accepted (they WANT it fixed NOW)
- [ ] Purchasing power verified: they have money OR can access it (credit, loans, partners)
- [ ] Targeting channels identified: where do they gather online/offline?
- [ ] Market trend validated: growing or at minimum stable (never declining)
- [ ] Competition exists (validates demand) but no dominant category king
- [ ] Willingness to pay verified via existing purchases in adjacent offers

**STOP CHECK:** If average score < 7/10, do NOT proceed to offer creation. Choose a different market or sub-segment.

## Section 2: Value Equation Audit

Score each variable of the Value Equation independently.

```
VALUE = (Dream Outcome x Perceived Likelihood) / (Time Delay x Effort & Sacrifice)
```

### Numerator Variables (MAXIMIZE)

| Variable | Current Score (1-10) | Target Score | Gap |
|----------|---------------------|--------------|-----|
| **Dream Outcome** | ___/10 | >= 8 | |
| **Perceived Likelihood** | ___/10 | >= 7 | |

#### Dream Outcome Checklist

- [ ] Dream outcome stated in prospect's EXACT words (not yours)
- [ ] Outcome is SPECIFIC and measurable ("lose 20lbs" not "lose weight")
- [ ] Outcome includes a timeframe ("in 60 days" not "eventually")
- [ ] Outcome addresses the REAL desire (status, freedom, belonging, security)
- [ ] Outcome is ambitious enough to create excitement
- [ ] Outcome is achievable enough to be believed
- [ ] Prospect would describe this outcome as "life-changing" if achieved
- [ ] Outcome compared favorably to the best alternative in the market

#### Perceived Likelihood Checklist

- [ ] At least 3 proof elements present (case studies, testimonials, data)
- [ ] Proof includes SPECIFIC numbers (not vague "many people")
- [ ] Proof includes people SIMILAR to the prospect (same demographics, situation)
- [ ] Before/after contrast is vivid and verifiable
- [ ] Guarantee present that reduces risk (see Section 6)
- [ ] Methodology/mechanism explained clearly (prospect understands WHY it works)
- [ ] Credibility indicators present (credentials, experience, media mentions)
- [ ] Social proof at scale (number of customers/users/results)

### Denominator Variables (MINIMIZE)

| Variable | Current Score (1-10) | Target Score | Gap |
|----------|---------------------|--------------|-----|
| **Time Delay** | ___/10 | <= 3 (lower = better) | |
| **Effort & Sacrifice** | ___/10 | <= 3 (lower = better) | |

#### Time Delay Checklist

- [ ] Time to first result clearly stated
- [ ] Time to full result clearly stated
- [ ] Quick wins designed into the first 7 days
- [ ] Progress milestones defined at regular intervals
- [ ] Speed compared to alternatives: "X% faster than [alternative]"
- [ ] If time delay is long, it is JUSTIFIED with a reason-why
- [ ] Onboarding designed to produce immediate small wins
- [ ] "Before today is over, you'll have [quick win]" element present

#### Effort & Sacrifice Checklist

- [ ] Level of effort honestly assessed and communicated
- [ ] "Done-for-you" elements identified and offered where possible
- [ ] "Done-with-you" elements have clear step-by-step instructions
- [ ] Sacrifice requirements minimized: what do they NOT have to give up?
- [ ] Compared to effort of alternatives: "Unlike [alternative], you don't have to..."
- [ ] Tools, templates, or automations provided to reduce manual work
- [ ] Support/community available for when they get stuck
- [ ] If effort is high, it is REFRAMED as investment, not sacrifice

### Value Equation Overall Score

```
Dream Outcome: ___/10
Perceived Likelihood: ___/10
─────────────────────────────
Numerator Score: ___ x ___ = ___

Time Delay: ___/10 (inverted: lower = more value)
Effort & Sacrifice: ___/10 (inverted: lower = more value)
─────────────────────────────
Denominator Score: ___ x ___ = ___

VALUE RATIO: Numerator / Denominator = ___
```

**Scoring guide:**
- Ratio >= 20: Grand Slam territory -- proceed with confidence
- Ratio 10-19: Strong offer, but optimize weakest variable
- Ratio 5-9: Mediocre offer -- significant work needed before launch
- Ratio < 5: DO NOT LAUNCH -- fix the fundamentals first

## Section 6: Guarantee Architecture

### Guarantee Type Selection

| Type | Using? | Description | Power Level |
|------|--------|-------------|-------------|
| Unconditional | [ ] | Money back, no questions, within X days | Moderate |
| Conditional | [ ] | Money back IF [specific condition] not met | Strong |
| Outcome-Based | [ ] | Pay only if [result] achieved | Very Strong |
| Anti-Guarantee | [ ] | All sales final (confidence signal) | Contrarian |

**Minimum requirement:** At least ONE guarantee present. Stacking recommended.

### Guarantee Quality Checklist

- [ ] Guarantee has a NAME (e.g., "The Iron-Clad Results Guarantee")
- [ ] Timeframe is specific (30, 60, 90, 180 days -- not "reasonable time")
- [ ] Guarantee is tied to a MEASURABLE outcome when conditional
- [ ] Guarantee is GENEROUS (exceeds what competitors offer)
- [ ] Reason-why for generosity is provided ("I can offer this because...")
- [ ] Guarantee is positioned as a SELLING ARGUMENT, not just risk removal
- [ ] Guarantee is placed next to the CTA (reinforces at decision point)
- [ ] Guarantee terms are simple and clear (no lawyer-speak)
- [ ] Guarantee is restated in the P.S. or closing section

### Stacked Guarantee Template

```
Layer 1 (Unconditional): "Try it for [X] days. If you don't love it
for ANY reason, just [action] and we'll refund every penny. No
questions. No hassle. You don't even need a reason."

Layer 2 (Conditional): "PLUS: If you [specific action steps] and
don't get [specific measurable result] within [timeframe], we'll
refund your money AND [additional compensation]."
```

- [ ] Both layers present and complementary
- [ ] Unconditional layer has shorter timeframe (creates safety net)
- [ ] Conditional layer has longer timeframe (requires commitment)
- [ ] Conditional layer requires specific actions (filters out non-serious)
- [ ] Combined guarantee makes the purchase feel RISK-FREE

## Section 7: Lead Generation - Core Four Audit

Evaluate your lead generation across all four channels.

### Channel Assessment

| Channel | Active? | Volume (weekly) | Quality (1-10) | CAC |
|---------|---------|-----------------|-----------------|-----|
| Warm Outreach | [ ] | ___ contacts | ___/10 | $____ |
| Content | [ ] | ___ pieces | ___/10 | $____ |
| Cold Outreach | [ ] | ___ contacts | ___/10 | $____ |
| Paid Ads | [ ] | ___ impressions | ___/10 | $____ |

**Minimum requirement:** At least 2 channels active. At least 1 channel profitable.

### Warm Outreach Audit

- [ ] Contact list of 100+ people who know you identified
- [ ] Outreach script uses "Who do you know...?" format (not direct pitch)
- [ ] Follow-up sequence defined (3-5 touches)
- [ ] Tracking: contacts made, conversations, referrals, conversions
- [ ] Volume target: minimum 10 warm contacts per day
- [ ] Each contact asked for REFERRALS even if not interested themselves
- [ ] Results logged weekly (contacts, conversations, bookings, closes)

### Content Audit

- [ ] Flagship platform chosen (YouTube, podcast, blog, newsletter)
- [ ] Publishing cadence defined and maintained (minimum 3x/week)
- [ ] Content delivers maximum "value per second"
- [ ] Content teaches WHAT and WHY; sells the HOW
- [ ] Content repurposed across 3+ platforms
- [ ] CTA included in every piece of content
- [ ] Engagement metrics tracked (views, comments, shares, leads)
- [ ] Content quality: "Would someone pay for this?" test passes

### Cold Outreach Audit

- [ ] Target list of 1,000+ prospects built
- [ ] Personalization strategy defined (not mass-blast)
- [ ] Multi-channel approach (email + LinkedIn + phone)
- [ ] Follow-up cadence: 5-9 touches over 30 days
- [ ] Volume target: minimum 50 cold contacts per day
- [ ] Response rate tracked and optimized (target: 5-15%)
- [ ] Booking rate tracked (target: 20-40% of responses)
- [ ] Scripts tested and iterated monthly

### Paid Ads Audit

- [ ] At least 1 platform active with spend > $50/day
- [ ] Creative testing volume: minimum 5 new creatives per week
- [ ] Ad-to-landing-page congruence verified
- [ ] CAC is <= 1/3 of first-purchase value
- [ ] ROAS tracked daily (target: >= 3x within 30 days)
- [ ] Winning creatives scaled; losers cut within 48 hours
- [ ] Landing page conversion rate >= 20% for lead magnets
- [ ] Retargeting active on all platforms

### Leverage Assessment

| Channel | You Do It | Others Do It For You |
|---------|-----------|---------------------|
| Warm Outreach | [ ] | [ ] Customers refer / Employees reach out |
| Content | [ ] | [ ] Team repurposes / Guests create |
| Cold Outreach | [ ] | [ ] Team / Agency handles volume |
| Paid Ads | [ ] | [ ] Agency manages / Affiliates run ads |

- [ ] At least 1 channel has "others do it" leverage active
- [ ] Referral program in place (customers generate warm leads)
- [ ] Affiliate/partner program considered or active
- [ ] Team members trained on at least 1 Core Four channel

## Section 9: Price-to-Value Ratio

### Pricing Framework Check

- [ ] Price is based on VALUE of result, not cost to deliver
- [ ] Price compared to cost of the PROBLEM (annualized/lifetime)
- [ ] Price compared to cost of ALTERNATIVES (professional fees, competitor prices)
- [ ] Daily cost breakdown calculated ("less than $X.XX/day")
- [ ] "Ferrari for $50" test: does the perceived value make the price feel trivial?
- [ ] Price is in the TOP 20% of the market (not competing on low price)

### Price Presentation Sequence

| Step | Element | Present? |
|------|---------|----------|
| 1 | Full value stack presented FIRST | [ ] |
| 2 | Total perceived value stated ($X,XXX) | [ ] |
| 3 | Compared to cost of the PROBLEM | [ ] |
| 4 | Compared to cost of ALTERNATIVES | [ ] |
| 5 | Daily/weekly cost breakdown | [ ] |
| 6 | "Your investment today: just $XXX" (contrast) | [ ] |
| 7 | Payment plan offered alongside single-pay | [ ] |
| 8 | Single-pay positioned as the SMART choice | [ ] |

- [ ] Price is NEVER mentioned before the full value stack
- [ ] "Just" or "only" used before the actual price amount
- [ ] No apology for the price ("This isn't cheap. Here's why.")
- [ ] Payment plan total is 10-20% more than single payment
- [ ] Payment plan amount compared to trivial daily expense
- [ ] Price justified by the VALUE EQUATION, not by cost breakdown

### Never-Discount Check

- [ ] No discount offered at any point in the funnel
- [ ] If price resistance encountered, BONUSES added (not price reduced)
- [ ] No coupon codes, flash sales, or "limited-time pricing"
- [ ] If a discount exists, it has a REAL logistic reason (launch pricing, beta)
- [ ] Discount (if any) is explicitly framed as one-time and non-repeating

## Section 11: Scarcity & Urgency Integrity

### Scarcity Audit

| Type | Using? | Is It REAL? | Logistic Reason? |
|------|--------|-------------|------------------|
| Limited quantity | [ ] | [ ] | [ ] "We can only serve X because..." |
| Limited time | [ ] | [ ] | [ ] "This offer expires on [date] because..." |
| Limited bonuses | [ ] | [ ] | [ ] "Only the first X get [bonus] because..." |
| Waitlist / Cohort | [ ] | [ ] | [ ] "Next cohort starts [date], capacity X" |

- [ ] Every scarcity claim has a LOGISTIC reason (not just "limited spots!")
- [ ] Exact numbers used ("47 remaining" not "limited")
- [ ] Scarcity is REAL and will be ENFORCED (not extended after deadline)
- [ ] Scarcity is a CONSEQUENCE of quality/service level (not arbitrary)
- [ ] No countdown timers that reset when the page is reloaded

### Urgency Audit

- [ ] Urgency has a REASON connected to the prospect (not just "act now")
- [ ] Deadline is specific: date + time (not "limited time")
- [ ] Consequence of missing deadline is EXPLICIT
- [ ] Urgency escalates through the sales process
- [ ] No fake urgency (Hormozi explicitly condemns this)
- [ ] If no real scarcity/urgency exists, NONE is used (honesty > manipulation)

## Quick Reference: The Hormozi Decision Tree

```
Is the market starving? (Section 1)
├── NO → Pick a different market
└── YES → Does the Value Equation ratio >= 10? (Section 2)
    ├── NO → Fix the weakest variable first
    └── YES → Are 5+ obstacles mapped to solutions? (Section 3)
        ├── NO → Map more obstacles from VOC data
        └── YES → Is the stack 10:1 value-to-price? (Section 4)
            ├── NO → Add bonuses or raise perceived value
            └── YES → Is there a named guarantee? (Section 6)
                ├── NO → Add guarantee (stacked recommended)
                └── YES → Are 2+ Core Four channels active? (Section 7)
                    ├── NO → Activate warm outreach first, then content
                    └── YES → LAUNCH. Volume negates luck.
```
