## Section 1: Mechanism Existence & Identity (10 items)

### 1.1 Does the mechanism EXIST?

- [ ] A unique mechanism is defined (not just a product feature or benefit)
- [ ] The mechanism has a SPECIFIC NAME (Sexy Cause for MUP, Gimmick Name for MUS)
- [ ] The mechanism is clearly ONE of 3 types:

| Type | Defined? | Name |
|------|----------|------|
| Ingredient (substance/compound) | [ ] | ___ |
| Process (method/sequence/protocol) | [ ] | ___ |
| Technology (tool/system/platform) | [ ] | ___ |

- [ ] The mechanism type is clearly identified (only ONE type per offer)

### 1.2 Can it be articulated?

- [ ] The mechanism can be explained in 1-2 sentences to a 12-year-old
- [ ] The explanation does NOT require technical jargon to understand
- [ ] Someone hearing it for the first time would say "Oh, I get it"

### 1.3 Is the name sticky?

- [ ] **Memorable** -- Can someone repeat the name 24 hours later without notes?
- [ ] **Intriguing** -- Does the name create a "what is that?" reaction?
- [ ] **Connected** -- Is the name obviously linked to the hero ingredient/process?
- [ ] **Speakable** -- Can someone tell a friend about it in conversation?
- [ ] **Not generic** -- Could a competitor use this exact name for THEIR product?

## Section 2: Mechanism Uniqueness (5 items)

### 2.1 Logo Test (Anti-Commodity)

Take the mechanism claim. Replace your product name with a competitor's.

- [ ] Does the claim STILL work with competitor's name?
  - [ ] YES = **FAIL.** Rewrite with mechanism specificity.
  - [ ] NO = **PASS.** Mechanism is unique.

### 2.2 Competitor Scan

- [ ] No direct competitor is using this mechanism (or a similar one)
- [ ] No competitor is using a mechanism with the same NAME
- [ ] The mechanism is PROPRIETARY -- only your product/method addresses it
- [ ] The mechanism cannot be easily reverse-engineered or copied by competitors

### 2.3 "So What?" Ladder

Start with the surface claim. Ask "So what?" three times.
The mechanism should live at level 3.

```
Level 1 (Feature):   "Our formula contains CoQ10"
                     So what? ↓
Level 2 (Benefit):   "It supports cellular energy"
                     So what? ↓
Level 3 (Mechanism): "It fuels the cochlear cells that otolith crystals
                      have been starving, allowing them to regenerate
                      for the first time in years"
```

- [ ] The mechanism claim is at Level 3 (not Level 1 or 2)

**Write your 3-level ladder:**
```
Level 1: ___
Level 2: ___
Level 3: ___
```

## Section 3: The 5 Functions of Mechanism (5 items)

Every mechanism must perform ALL 5 functions. A mechanism missing
any function is incomplete and will underperform.

| # | Function | Present? | How It Does This |
|---|----------|----------|-----------------|
| 1 | **EXPLAIN** -- Why the problem exists | [ ] | ___ |
| 2 | **ABSOLVE** -- Remove blame from prospect | [ ] | ___ |
| 3 | **INVALIDATE** -- Why other solutions failed | [ ] | ___ |
| 4 | **POSITION** -- Why YOUR solution is different | [ ] | ___ |
| 5 | **URGENTIZE** -- Why they must act NOW | [ ] | ___ |

**Function completeness score:** ___/5

### Critical Check: Absolve Function

The absolve function is the most commonly missed. Without it, the
prospect carries guilt/blame into the sales conversation and resists
buying ("I should have done X differently").

- [ ] The mechanism explicitly removes blame from the prospect
- [ ] The blame is redirected to an EXTERNAL cause (not the prospect's behavior)
- [ ] The external cause is SPECIFIC (not vague: "toxins" or "bad advice")

## Section 4: Paradigm Shift Quality (5 items)

The mechanism must CREATE a paradigm shift. If the prospect can fit
the mechanism into their EXISTING worldview without changing anything
they believe, the shift has not occurred.

### 4.1 Paradigm Shift Architecture

| Step | Defined? | Content |
|------|----------|---------|
| CURRENT PARADIGM (what they believe now) | [ ] | ___ |
| DISRUPTION (why that belief is incomplete) | [ ] | ___ |
| NEW PARADIGM (the new understanding) | [ ] | ___ |
| BRIDGE TO SOLUTION (how product fits) | [ ] | ___ |

### 4.2 Shift Quality Tests

- [ ] The old belief is REAL and SPECIFIC (not a straw man nobody actually holds)
- [ ] The disruption feels like a REVELATION, not a marketing argument
- [ ] The new paradigm feels INEVITABLE once the disruption is presented
- [ ] The new paradigm EXPLAINS past failures ("Oh, THAT'S why nothing worked")
- [ ] The bridge to solution is NATURAL (not forced -- the paradigm DEMANDS this type of solution)

### 4.3 Emotional Temperature

After reading the paradigm shift, does the prospect feel:
- [ ] SMARTER (not dumber or insulted)
- [ ] HOPEFUL (not just scared)
- [ ] CURIOUS about the solution (not just worried about the problem)

## Section 5: New Opportunity Positioning (5 items)

The mechanism must frame the offer as a NEW OPPORTUNITY, not an
improvement on something that already exists.

### 5.1 Category Test

- [ ] The offer is described WITHOUT referencing any existing category
  - FAIL: "A better supplement for tinnitus"
  - PASS: "A 4-ingredient otolith crystal protocol"

- [ ] The mechanism creates its OWN category that didn't exist before
- [ ] No "better than" or "improved" language is used anywhere
- [ ] The prospect cannot mentally file this offer with previous failures
- [ ] The offer feels like something they've NEVER HAD ACCESS TO before (not something they tried and it didn't work)

### 5.2 Failure Immunity Test

```
Prospect's internal dialogue:
"I tried something like this before and it didn't work."

Does the mechanism PREVENT this thought?
[ ] YES -- The mechanism makes it clear this is fundamentally different
[ ] NO  -- The prospect could see this as "another [category]"
```

If NO, the new opportunity framing needs work.

## Section 6: Visual & Tangibility (5 items)

The mechanism must be VISUAL. The prospect must be able to SEE it
in their mind. Abstract mechanisms fail because the brain cannot
form belief around something it cannot picture.

- [ ] The mechanism describes something PHYSICAL or VISIBLE (even if microscopic)
- [ ] A non-expert could DRAW the mechanism if asked (even crudely)
- [ ] The mechanism uses CONCRETE language (not abstract: "energy," "balance," "alignment")
- [ ] The mechanism involves a SPECIFIC body part, process, or system (not "the body" or "the brain")
- [ ] The mechanism action is describable with a VERB the prospect can visualize (dissolve, decode, activate, unblock, rewire)

**Visualization Test:**
Describe the mechanism as if to a blind person. Can they picture it?

```
___
```

## Section 9: Revelation Sequence (5 items)

The mechanism must be REVEALED, not dumped. The revelation sequence
builds anticipation and makes the mechanism land with maximum impact.

| Step | Present? | Content |
|------|----------|---------|
| 1. **TEASE** -- Hint that a mechanism exists | [ ] | ___ |
| 2. **CONTEXT** -- Set up why this matters (historical, authority) | [ ] | ___ |
| 3. **REVEAL** -- Name and explain the mechanism | [ ] | ___ |
| 4. **VALIDATE** -- Prove the mechanism is real (study, expert) | [ ] | ___ |
| 5. **CONNECT** -- Bridge mechanism to YOUR solution | [ ] | ___ |

- [ ] The sequence creates a "building" feeling (each step increases engagement)
- [ ] The reveal does NOT happen before tease + context (no premature revelation)
- [ ] Validation comes AFTER reveal, not before (prove what you just claimed)
- [ ] Connection feels NATURAL (the mechanism DEMANDS this type of solution)
