# Kennedy Close Stack Checklist

> **Expert:** Dan Kennedy
> **Source:** The Ultimate Sales Letter, No B.S. Direct Marketing, Magnetic Marketing, No B.S. Price Strategy
> **Use:** Build and validate a Kennedy-quality close. Every item represents a specific Kennedy principle.

## Section 1: Close Architecture (7 Layers)

Every Kennedy close is built in layers. Check each layer is present.

| Layer | Element | Present? | Notes |
|-------|---------|----------|-------|
| 1 | **Summary Close** — restate all components with values | [ ] | |
| 2 | **Value Stack** — total value shown before price (10:1 gap) | [ ] | |
| 3 | **If/Then Close** — purchase framed as logical conclusion | [ ] | |
| 4 | **Reason-Why Close** — every scarcity/deadline element explained | [ ] | |
| 5 | **Risk Reversal** — named guarantee next to CTA | [ ] | |
| 6 | **Deadline Stack** — 2+ urgency layers with progressive reveal | [ ] | |
| 7 | **Final CTA + PS Sequence** — imperative action + 2-3 PS sections | [ ] | |

**Minimum: 5/7 layers present. Below 5 = REBUILD the close.**

## Section 2: Urgency Stack (4 Types)

At least 2 types MUST be active. Each MUST have a reason-why.

| Type | Active? | Specific? | Has Reason-Why? | Restated 3x? |
|------|---------|-----------|-----------------|--------------|
| TIME (calendar deadline with date/time) | [ ] | [ ] | [ ] | [ ] |
| QUANTITY (limited supply with exact number) | [ ] | [ ] | [ ] | [ ] |
| BONUS (disappearing addition with removal date) | [ ] | [ ] | [ ] | [ ] |
| PRICE (increasing cost with future amount stated) | [ ] | [ ] | [ ] | [ ] |

- [ ] At least 2 urgency types active
- [ ] Deadlines use SPECIFIC dates and times (not "limited time" or "hurry")
- [ ] Each deadline has a LOGISTIC reason (not "because we care")
- [ ] Deadlines are revealed PROGRESSIVELY (not all dumped at once)
- [ ] Consequence of missing each deadline is EXPLICITLY stated
- [ ] All deadlines restated at least 3 times across the close

## Section 3: Guarantee Structure

| Criterion | Check | Detail |
|-----------|-------|--------|
| Guarantee is NAMED? | [ ] | Name: _________________________ |
| Tied to MEASURABLE outcome? | [ ] | Outcome: _________________________ |
| Has SPECIFIC timeframe? | [ ] | Timeframe: _________________________ |
| Has reason-why? | [ ] | "I can offer this because..." |
| Placed NEXT to CTA? | [ ] | |
| At least Level 2 strength? | [ ] | |
| Guarantee RESTATES the main benefit? | [ ] | |

**Guarantee Power Level:**
- [ ] Level 1: "Money back if not satisfied" -- WEAK (invisible)
- [ ] Level 2: "Full refund, no questions, keep bonuses" -- MODERATE
- [ ] Level 3: "Double money back if [result] doesn't happen" -- STRONG
- [ ] Level 4: "[Result] or I pay you $[amount]" -- NUCLEAR
- [ ] Level 5: "Better-than-money-back: refund + keep everything + compensation" -- SUPREME

**Minimum: Level 2. Target: Level 3+.**

## Section 4: Price Presentation

| Technique | Used? | Detail |
|-----------|-------|--------|
| **Cost of the Problem** — quantified what the problem costs them | [ ] | $ amount: _______ |
| **Cost of Alternatives** — compared to what they would spend elsewhere | [ ] | Alternative: _______ |
| **Daily Cost Breakdown** — price reduced to daily/trivial amount | [ ] | Per day: _______ |
| **Value Stack Before Reveal** — all components listed with individual values | [ ] | Total stack: _______ |
| **Payment Plan Reframe** — payments framed as trivial daily/weekly cost | [ ] | Per payment: _______ |

- [ ] Price is presented AFTER value stack (never lead with price)
- [ ] Total stacked value is at minimum 10:1 vs. actual price
- [ ] Each component in the stack has a SPECIFIC dollar value attached
- [ ] Price is compared to at least ONE alternative/cost-of-problem anchor
- [ ] No apology for price ("This isn't cheap. Here's why it shouldn't be.")
- [ ] Daily cost compared to a trivial expense (coffee, parking, etc.)

## Section 5: Reason-Why Audit

EVERY offer element needs a reason. Check each one.

| Offer Element | Has Reason-Why? | Sounds Like Confession? | Logistic/Real? |
|---------------|-----------------|------------------------|----------------|
| Price (why this amount) | [ ] | [ ] | [ ] |
| Deadline (why it expires) | [ ] | [ ] | [ ] |
| Guarantee (why so generous) | [ ] | [ ] | [ ] |
| Each bonus (why included) | [ ] | [ ] | [ ] |
| Discount if any (why the drop) | [ ] | [ ] | [ ] |
| Exclusivity if any (why not everyone) | [ ] | [ ] | [ ] |
| Free content if any (why giving away) | [ ] | [ ] | [ ] |

- [ ] ZERO unexplained offer elements
- [ ] Reasons sound like confessions, not marketing
- [ ] Reasons are grounded in logistics, circumstance, or business reality
- [ ] No altruistic reasons ("because we care" = suspicious)
- [ ] One good reason per element (not three mediocre ones)

## Section 6: CTA Stack

| CTA Layer | Purpose | Present? |
|-----------|---------|----------|
| CTA #1 | WHAT to do ("Click the button below right now") | [ ] |
| CTA #2 | HOW to do it ("Fill in name, select package, done") | [ ] |
| CTA #3 | WHAT HAPPENS when they do ("Within 60 seconds, you'll receive...") | [ ] |
| CTA #4 | CONSEQUENCE of NOT doing ("If you leave, this offer disappears") | [ ] |
| CTA #5 | IF/THEN close ("If you're serious about X, then Y is the only logical next step") | [ ] |

- [ ] Minimum 3 CTA layers present
- [ ] Imperative verbs used: "Click", "Call", "Enter", "Select"
- [ ] Process described step-by-step (absurdly specific)
- [ ] Immediate aftermath painted (what happens in next 60 seconds)
- [ ] Button text is SPECIFIC ("Send Me [Product] Now" not "Submit")
- [ ] Negative CTA present (what they LOSE by not acting)

## Section 7: PS Sequence

| PS Type | Present? | Content Check |
|---------|----------|---------------|
| **Benefit PS** — restates strongest benefit | [ ] | References specific result |
| **Deadline PS** — restates hardest urgency element | [ ] | Specific date/time |
| **Testimonial PS** — adds proof not in main copy | [ ] | Name + specific result |
| **New-Angle PS** — introduces benefit not yet covered | [ ] | Genuinely new info |
| **Fear PS** — consequence of inaction | [ ] | Specific loss stated |

- [ ] At least 2 PS sections present
- [ ] Each PS works as a STANDALONE ad (reader may only read these)
- [ ] PS sections numbered: P.S., P.P.S., P.P.P.S.
- [ ] First PS = strongest benefit OR strongest proof
- [ ] Last PS = hardest deadline OR strongest consequence
- [ ] No PS longer than 3-4 sentences

## Section 8: Trial Closes

Trial closes should appear THROUGHOUT the copy, not just at the end.

- [ ] Early trial close present (after problem section): "If this sounds like you..."
- [ ] Middle trial close present (after mechanism): "Sound like something worth trying?"
- [ ] Pre-close present (after proof): "By now, one of two things is true..."
- [ ] If/Then close used at least once in the close section
- [ ] Each trial close builds commitment toward the final CTA

---

## Section 9: Takeaway / Disqualification

- [ ] "This isn't for everyone" statement present
- [ ] At least 2 specific disqualification criteria listed
- [ ] Disqualification positioned BEFORE the CTA (builds desire)
- [ ] Qualification criteria make ideal buyer feel SELECTED
- [ ] Tone is confident, not apologetic ("I don't need everyone. I need the RIGHT people.")

---

## Section 10: Damaging Admission

- [ ] At least ONE genuine weakness or limitation admitted
- [ ] Placed BEFORE the close (builds trust for the ask)
- [ ] Weakness REFRAMED as filter or feature
- [ ] Does NOT undermine the core promise
- [ ] Feels genuinely honest, not manufactured vulnerability
- [ ] Strengthens positioning: "It's [weakness] BECAUSE [reframe]"

## Quick Reference: Kennedy Close Non-Negotiables

These 5 items are NEVER optional:

1. **Reason-why for every scarcity element.** No exceptions.
2. **Named guarantee next to CTA.** Unnamed guarantee = invisible.
3. **2+ urgency layers with specific dates/numbers.** Vague urgency = no urgency.
4. **3+ CTA layers.** One CTA = one chance. Three CTAs = three chances.
5. **2+ PS sections.** Each PS is a second headline. Waste none.
