# Ry Schwartz Conversion Copy -- Master Checklist

> **Source:** Ry Schwartz Voice DNA Profile
> **Use:** Validate any email sequence, launch campaign, or SaaS onboarding copy against Ry Schwartz's Coaching The Conversion principles before delivery.
> **Ref:** `~/.claude/copy-squad/ry-schwartz.md`

## 1. VOC Mining Quality Audit

### Data Collection

- [ ] VOC gathered from 3+ distinct sources (surveys, reviews, forums, interviews, support tickets)
- [ ] Minimum 100 data points collected per audience segment
- [ ] Sources include EMOTIONAL language, not just rational feedback
- [ ] Data recorded VERBATIM (not paraphrased or cleaned up)
- [ ] Each quote tagged with: source platform, emotional intensity (1-5), awareness stage
- [ ] Repeated phrases flagged (frequency = weight)
- [ ] Unusual metaphors and vivid descriptions highlighted
- [ ] Gap between what customers SAY vs what they DO acknowledged

### Iceberg Analysis (Deep Interpretation)

- [ ] Analyst spent time SITTING WITH the data (not just skimming)
- [ ] Surface meanings identified (what they said)
- [ ] Deeper meanings interpreted (what led them to say it)
- [ ] Unstated fears documented (what they did NOT say but clearly feel)
- [ ] Desires behind desires identified (surface desire vs hidden desire vs inadmissible desire)
- [ ] Identity-level pain uncovered (who they fear they ARE, not just what they experience)
- [ ] Patterns across sources triangulated (same theme from 3+ sources = high confidence)

### VOC Deployment in Copy

- [ ] Headlines use VERBATIM customer language (not marketing translation)
- [ ] Lead paragraphs mirror the reader's internal dialogue
- [ ] Objection-handling sections use the reader's own words for their doubts
- [ ] CTA language reflects how the customer describes the desired outcome
- [ ] Parenthetical asides and PS sections use exact VOC phrases
- [ ] No marketing-speak substituted where customer language exists
- [ ] VOC language appears in EVERY major section (not just the lead)

### VOC Quality Scale

| Score | Description | Action |
|-------|-------------|--------|
| 1 | No VOC used -- copy is pure imagination | REWRITE from scratch with VOC foundation |
| 2 | Some VOC language sprinkled in headlines only | EXPAND VOC use throughout all sections |
| 3 | VOC in headlines and lead, marketing-speak elsewhere | REPLACE marketing-speak with customer language |
| 4 | VOC throughout, but surface-level (no iceberg analysis) | DEEPEN interpretation -- what is UNDERNEATH? |
| 5 | Deep VOC throughout, but single-source | ADD triangulation from 2+ additional sources |
| 6 | Multi-source VOC, deep interpretation, but some sections generic | IDENTIFY and FIX remaining generic sections |
| 7 | Strong VOC foundation across all sections | ACCEPTABLE -- refine details |
| 8 | Deep, multi-source, well-deployed VOC with iceberg insights | GOOD -- minor polish only |
| 9 | VOC so precise the reader says "how did you know?" | EXCELLENT -- ready for production |
| 10 | Indistinguishable from a transcript of the reader's inner monologue | EXCEPTIONAL -- deploy immediately |

**Minimum passing score: 7**

## 2. Email Sequence Architecture Check

### Pre-Sequence Planning

- [ ] Full sequence mapped BEFORE writing the first email
- [ ] Each email has a defined STARTING awareness stage
- [ ] Each email has a defined ENDING awareness stage
- [ ] No awareness stage is skipped in the progression
- [ ] Each email has a single BELIEF SHIFT defined
- [ ] Belief progression is sequential (each belief BUILDS on the previous)
- [ ] Emotional state transitions tracked email-to-email
- [ ] Total sequence length justified by the awareness gap to bridge

### Per-Email Validation

For EACH email in the sequence, verify:

- [ ] Starting awareness stage matches ending stage of previous email
- [ ] ONE primary belief shift per email (not multiple)
- [ ] Belief shift is EARNED by the content (not asserted)
- [ ] Opening joins the reader's current conversation (not the writer's agenda)
- [ ] Body advances the reader toward the ending awareness stage
- [ ] CTA matches the ending awareness level (no Product Aware CTA in a Problem Aware email)
- [ ] ONE CTA maximum (no competing links or actions)
- [ ] PS reinforces CTA or plants seed for next email
- [ ] Email provides standalone value (worth reading even without the CTA)
- [ ] Email answers both universal objections: "What's in it for me?" and "Do I want to stay subscribed?"

### Sequence Structural Integrity

- [ ] Sequence begins by joining the reader's EXISTING conversation
- [ ] Problem is deepened BEFORE solution is introduced
- [ ] Mechanism/solution is introduced THROUGH story (not alongside it)
- [ ] Objections are handled across multiple emails (not all in one)
- [ ] Social proof escalates through the sequence (not front-loaded)
- [ ] Final emails coach DECISIVENESS, not urgency
- [ ] Voice is consistent across all emails (one person, not a committee)
- [ ] Length is appropriate per email type (200-600 words, varies by purpose)

### Email Architecture Quality Scale

| Score | Description | Action |
|-------|-------------|--------|
| 1 | No awareness mapping -- emails written in isolation | MAP the full sequence from scratch |
| 2 | Awareness stages defined but skipped in execution | IDENTIFY gaps and add bridging emails |
| 3 | Sequential awareness but no belief tracking | ADD belief before/after for each email |
| 4 | Beliefs tracked but not earned (assertions, not coaching) | REWRITE to EARN each belief shift through story/proof |
| 5 | Good structure but multiple CTAs or awareness mismatches | FIX CTA alignment and remove competing actions |
| 6 | Solid architecture, minor belief progression gaps | TIGHTEN the weakest belief transitions |
| 7 | Strong architecture, consistent voice, good flow | ACCEPTABLE -- refine details |
| 8 | Professional-grade sequence with strong coaching flow | GOOD -- minor polish only |
| 9 | Each email feels like a personal coaching session | EXCELLENT -- ready for production |
| 10 | Reader feels they discovered the decision on their own | EXCEPTIONAL -- deploy immediately |

**Minimum passing score: 7**

## 3. Awareness-Stage Alignment Audit

### Stage Identification

For the target audience, verify:

- [ ] Starting stage of awareness CORRECTLY identified (not assumed)
- [ ] Evidence supports the starting stage assessment (behavioral data, survey, segment behavior)
- [ ] Ending stage required for conversion identified (usually Most Aware or Product Aware)
- [ ] Number of stages to bridge calculated (determines minimum sequence length)

### Per-Stage Content Audit

**UNAWARE -> PROBLEM AWARE Emails:**

- [ ] Opens with a story or scenario that mirrors the reader's situation
- [ ] Does NOT name the problem directly (let the reader recognize it)
- [ ] Creates a "wait, is this happening to me?" moment
- [ ] Uses identification as the primary emotional lever
- [ ] ZERO product mention
- [ ] ZERO solution mention

**PROBLEM AWARE -> SOLUTION AWARE Emails:**

- [ ] Deepens problem awareness through empathetic challenge
- [ ] Uses the Bieber Template or similar apology-based approach where appropriate
- [ ] Introduces the MECHANISM (why the problem exists), not the product
- [ ] Explains why previous attempts failed (not the reader's fault)
- [ ] Validates the reader's frustration and past experiences
- [ ] Creates openness for a new approach

**SOLUTION AWARE -> PRODUCT AWARE Emails:**

- [ ] Differentiates YOUR approach from alternatives clearly
- [ ] Mechanism fully revealed (why THIS solution is structurally different)
- [ ] Proof stacking begins (stories, data, expert validation)
- [ ] Product introduced through story (not a pitch)
- [ ] Reader understands WHY this solution addresses their specific situation

**PRODUCT AWARE -> MOST AWARE Emails:**

- [ ] Objections handled through narrative (not FAQ lists)
- [ ] Deep objections addressed (identity-level, not just surface)
- [ ] Social proof from people the reader identifies with
- [ ] Case studies show specific, measurable, emotional results
- [ ] Risk reversal presented as confidence signal
- [ ] Value stacking makes the offer feel obvious

**MOST AWARE -> BUYER Emails:**

- [ ] Tone is relaxed and confident (NOT panicked or pressure-filled)
- [ ] Decisiveness coached (not urgency manufactured)
- [ ] Reader feels SAFE making a decision
- [ ] CTA is simple, clear, and singular
- [ ] Story of someone who was in the exact same position and acted
- [ ] PS acknowledges that not buying is also okay (reduces pressure, paradoxically increases conversion)

### Awareness Mismatch Detector

| Sign | Likely Problem | Fix |
|------|---------------|-----|
| Low open rates on early emails | Subject lines too salesy for awareness level | Rewrite subjects to match starting awareness (curiosity, not offers) |
| High opens but low clicks | Content does not match stated awareness promise | Align email body with subject line's implied awareness level |
| Clicks but no conversions | CTA awareness mismatch (asking for Product Aware action from Solution Aware readers) | Add bridging emails before the conversion ask |
| Unsubscribes spike mid-sequence | Too aggressive too early (jumping awareness stages) | Slow down the progression, add value emails between pitches |
| High conversions but high refunds | Converted at wrong awareness level (bought before fully convinced) | Strengthen Product Aware emails to build deeper conviction |

## 4. Objection Handling Verification

### Surface Objection Coverage

- [ ] Price objection addressed (value framed, not discounted)
- [ ] Time objection addressed (implementation timeline clarified)
- [ ] "Does it work?" objection addressed (proof stacked)
- [ ] "Does it work for ME?" objection addressed (specificity, identification)
- [ ] "What if it doesn't work?" objection addressed (guarantee, risk reversal)
- [ ] "Is this the right time?" objection addressed (cost of waiting framed)

### Deep Objection Coverage (The Iceberg)

- [ ] Past failure shame addressed (story of someone who failed before succeeding HERE)
- [ ] Social judgment fear addressed (normalization of the decision)
- [ ] Identity doubt addressed ("Am I the kind of person who..." -- YES, and here's proof)
- [ ] Fear of hope addressed (acknowledged directly with empathy)
- [ ] "Too late/old/broken" belief addressed (case study of someone who started later and succeeded)
- [ ] "I won't follow through" fear addressed (structural support built into the offer)

### Objection Handling Method Audit

- [ ] Objections addressed through STORY (not bulleted FAQ)
- [ ] Each major objection has its own email (not crammed into one)
- [ ] Objections are ANTICIPATED (addressed before the reader voices them)
- [ ] Objections are treated as VALID (not dismissed or minimized)
- [ ] Handling uses social proof from relatable people (not just authority claims)
- [ ] Deep objections addressed with empathy, not logic
- [ ] No false empathy ("I know exactly how you feel" without demonstration)

### Objection Coverage Scale

| Score | Description | Action |
|-------|-------------|--------|
| 1 | No objection handling present | ADD dedicated objection-handling emails |
| 2 | Surface objections mentioned but not addressed through story | REWRITE using narrative-based handling |
| 3 | FAQ-style handling (bulleted, clinical) | RESTRUCTURE into story-driven emails |
| 4 | Story-based but only surface objections | ADD deep objection handling (identity-level) |
| 5 | Some deep objections addressed but not all | IDENTIFY missing deep objections and add coverage |
| 6 | Good coverage but some objections feel dismissed rather than honored | REVISE tone to validate objections as legitimate |
| 7 | Comprehensive coverage, story-driven, empathetic | ACCEPTABLE -- refine details |
| 8 | Deep and surface objections handled through compelling narrative | GOOD -- minor polish only |
| 9 | Reader feels their specific doubts were personally addressed | EXCELLENT -- ready for production |
| 10 | Objection handling so empathetic it builds trust beyond the sale | EXCEPTIONAL -- deploy immediately |

**Minimum passing score: 7**

## 5. Conversion Copy Scoring (Per-Email Audit)

### The Two Sweeps

**Sweep 1: Empathy Sweep (30 minutes)**

Read the entire piece from the prospect's perspective. For each section:

- [ ] Does this section show I UNDERSTAND the reader's situation? (Not assume it)
- [ ] Is the copy talking WITH the reader or AT them?
- [ ] Does it use VOC language or marketing-speak?
- [ ] Would the reader feel SEEN by this paragraph?
- [ ] Is there a specific detail that proves understanding (not a generic statement)?

Empathy Sweep Results:
- Sections that talk AT the reader: [ ] Identified and marked
- Generic marketing-speak: [ ] Identified and replaced with VOC
- Missing empathy demonstrations: [ ] Identified and added
- Assumptions without demonstration: [ ] Identified and earned

**Sweep 2: Belief Sweep (30 minutes)**

Read the entire piece and track belief progression. For each transition:

- [ ] Has the previous section EARNED the next claim?
- [ ] Is there a leap of logic between paragraphs?
- [ ] Are assertions supported by proof, story, or VOC?
- [ ] Is the belief progression sequential (no skipped steps)?
- [ ] Does each section advance ONE belief (not multiple)?

Belief Sweep Results:
- Unearned claims: [ ] Identified and supported with proof/story
- Logic leaps: [ ] Identified and bridged
- Belief gaps: [ ] Identified and filled
- Skipped steps: [ ] Identified and sequenced properly

**Email CTC Score = Average of all applicable criteria**

### Conversion Hierarchy Check

For the complete sequence, verify prioritization:

**HIGHEST IMPACT (Must be excellent):**
- [ ] Subject lines earn the open (score >= 8)
- [ ] Leads join the reader's conversation within 3 sentences (score >= 8)
- [ ] CTAs are clear, singular, and earned by the copy above (score >= 8)

**HIGH IMPACT (Must be strong):**
- [ ] Deep objections addressed through story (score >= 7)
- [ ] Proof stacked until doubt feels irrational (score >= 7)
- [ ] Copy speaks to the correct awareness stage (score >= 7)

**MODERATE IMPACT (Should be solid):**
- [ ] Transitions flow naturally between sections (score >= 6)
- [ ] Voice consistent across all emails (score >= 6)
- [ ] Length calibrated per email purpose (score >= 6)

**LOWER IMPACT (Polish):**
- [ ] Formatting scannable with short paragraphs (score >= 5)
- [ ] PS reinforces CTA or adds a hook (score >= 5)
- [ ] Preheader complements subject line (score >= 5)

## 6. Launch Sequence Audit

### Pre-Launch Phase (7-14 Days Before)

- [ ] Origin story sequence present (minimum 3 emails)
- [ ] First email establishes personal credibility through vulnerability
- [ ] Problem reframing introduced BEFORE the offer exists
- [ ] Mechanism teased but NOT fully revealed
- [ ] Open loops created that only the launch can close
- [ ] 70% of the persuasion heavy-lifting happens HERE
- [ ] Reader ends pre-launch believing their old approach was wrong

### Launch Phase (Days 0-3)

- [ ] Offer presented as logical extension of pre-launch education
- [ ] Mechanism fully revealed on launch day
- [ ] Social proof escalation begins (small proof -> big proof)
- [ ] First CTA appears naturally (not abruptly)
- [ ] Tone shifts from education to invitation (not from value to pitch)

### Sales Phase (Days 3-5)

- [ ] Case studies feature relatable people (not just impressive results)
- [ ] Each sales email handles ONE specific objection
- [ ] FAQ handled through narrative, not bullet points
- [ ] Hidden benefits revealed (unexpected value beyond the main promise)
- [ ] Social proof continues to escalate

### Closing Phase (Final 24-48 Hours)

- [ ] Tone is relaxed and confident (NOT frantic)
- [ ] Decisiveness coached, not urgency manufactured
- [ ] Story of someone who almost did not act (mirror the reader's position)
- [ ] Consequences of action vs inaction presented through story (not threat)
- [ ] Final CTA is simple and singular
- [ ] PS acknowledges the reader's autonomy in the decision
- [ ] Mood is lighter than expected (counterintuitive but effective)
- [ ] Reader should feel EMPOWERED to decide, not PRESSURED to buy

### Launch Sequence Quality Scale

| Score | Description | Action |
|-------|-------------|--------|
| 1 | No launch structure -- just sales emails | REDESIGN with full CTC launch architecture |
| 2 | Sales emails only, no pre-launch | ADD pre-launch origin story sequence |
| 3 | Pre-launch exists but does not shift beliefs | REWRITE pre-launch with specific belief targets |
| 4 | Good pre-launch but launch is abrupt (value -> pitch switch) | SMOOTH the transition, make offer feel natural |
| 5 | Solid structure but urgency-heavy closing | REPLACE urgency with decisiveness coaching |
| 6 | Good structure, some belief gaps in sales phase | TIGHTEN sales phase objection handling |
| 7 | Professional launch sequence with clear coaching flow | ACCEPTABLE -- refine details |
| 8 | Each phase feels distinct and purposeful | GOOD -- minor polish only |
| 9 | Reader feels they arrived at the buying decision on their own | EXCELLENT -- ready for production |
| 10 | Launch so well-coached the prospect thanks you regardless of buying | EXCEPTIONAL -- deploy immediately |

**Minimum passing score: 7**

## 7. SaaS Onboarding Audit

### Sequence Structure

- [ ] Three tracks present: essential onboarding, post-aha-moment, free-to-paid
- [ ] Triggers are behavior-based (not time-based)
- [ ] Each email focuses on ONE feature
- [ ] Feature is framed as solving ONE specific user pain
- [ ] Emails are substantive (3x longer than a typical product notification)

### Per-Email Feature Sell

- [ ] Opens with empathy (WHY the user signed up)
- [ ] Mirrors the user's goal before teaching the how-to
- [ ] ONE feature positioned as solving ONE problem
- [ ] Story of a user who activated this feature and saw improvement
- [ ] Instructions wrapped in benefit and narrative (not dry documentation)
- [ ] ONE CTA: use this feature NOW
- [ ] CTA action is simple and quick (under 5 minutes)

### Conversion Path

- [ ] Essential onboarding drives to the activation event
- [ ] Post-activation sequence deepens engagement
- [ ] Free-to-paid sequence frames each feature as a reason to stay
- [ ] Trial deadline used as natural urgency (no artificial scarcity)
- [ ] Focus on getting user to EXPERIENCE value (not fear losing access)
- [ ] Behavioral triggers advance users to appropriate next email

### SaaS Onboarding Quality Scale

| Score | Description | Action |
|-------|-------------|--------|
| 1 | Generic "welcome" email + countdown timer | REDESIGN with 3-track behavior-based architecture |
| 2 | Time-based drip with feature descriptions | REWRITE as feature sell emails with story and empathy |
| 3 | Feature-focused but dry/instructional tone | ADD empathy openers and user success stories |
| 4 | Good content but single-track (no behavioral branching) | ADD behavior-based triggers and multiple tracks |
| 5 | Multi-track but emails too short (notification-style) | EXPAND emails to sales-letter depth per feature |
| 6 | Solid structure, some emails lack benefit framing | REFRAME remaining features as pain-solving stories |
| 7 | Strong 3-track system with story-driven feature sells | ACCEPTABLE -- refine details |
| 8 | Behavior-based, empathetic, with clear activation path | GOOD -- minor polish only |
| 9 | Users feel guided, not marketed to | EXCELLENT -- ready for production |
| 10 | Trial users convert because they experienced value, not feared losing access | EXCEPTIONAL -- deploy immediately |

**Minimum passing score: 7**

### Blocker Rules (Override Score)

Regardless of total score, the following are AUTOMATIC BLOCKERS:

- [ ] VOC Mining Quality below 6 = BLOCKER (no deployment without real customer language)
- [ ] Awareness-Stage Alignment below 6 = BLOCKER (mismatched copy cannot convert)
- [ ] Any email with urgency-first closing (no decisiveness coaching) = BLOCKER
- [ ] Any email with FAQ-style objection handling (no narrative) = BLOCKER
- [ ] Any sequence with skipped awareness stages = BLOCKER
- [ ] Any email with multiple CTAs = BLOCKER
- [ ] Any pre-launch with zero belief-shifting content = BLOCKER

### Quick Audit (5-Minute Version)

For fast validation when full audit is not feasible:

- [ ] Is VOC language present in every email? (YES/NO)
- [ ] Is each email mapped to a specific awareness stage? (YES/NO)
- [ ] Does each email coach ONE belief shift? (YES/NO)
- [ ] Are objections handled through story, not FAQ? (YES/NO)
- [ ] Does the closing email coach decisiveness, not urgency? (YES/NO)
- [ ] Is each CTA singular, clear, and earned by the copy above? (YES/NO)
- [ ] Does the copy feel like coaching, not marketing? (YES/NO)

**If any answer is NO, that area requires attention before deployment.**
