# Task: Offer Stack Construction

> **Primary Expert:** Todd Brown (architecture), Dan Kennedy (urgency/guarantee)
> **Supporting:** Gary Halbert (bullets for each component)

---

## Structure

```
CORE PRODUCT       - Delivers the mechanism directly
COMPONENT 2        - Accelerates mechanism results
COMPONENT 3        - Removes #1 obstacle to mechanism working
COMPONENT 4        - Addresses secondary concern tied to mechanism
BONUS (conditional) - Quick win that proves mechanism works
GUARANTEE          - Named, specific, measurable (Kennedy)
```

## Per-Component Template

```
NAME: [Mechanism-specific name, NOT generic]
WHAT: [What it is, one sentence]
WHY: [How it connects to the mechanism]
VALUE: [Odd number: $197, $347, not $200, $350]
BULLET: [Halbert-style fascination bullet for this component]
```

## Quality Checklist

- [ ] Every component connects to mechanism (no filler)
- [ ] All names use mechanism-specific language
- [ ] Values are odd numbers
- [ ] Value multiple is 10x+ of asking price
- [ ] Core product is clearly the most valuable
- [ ] Guarantee is named and specific (Kennedy Level 2+)
- [ ] Each component has a Halbert-style fascination bullet
- [ ] Anti-Commodity Test: competitor cannot copy the stack
