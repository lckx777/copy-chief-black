# RMBC Framework
## Reason, Mechanism, Believability, Connection

Framework de validação para copy de Direct Response. Todo mecanismo (MUS) deve passar nos 4 critérios.

---

## Os 4 Pilares

### R - Reason Why (Por que funciona)

**Definição:** A explicação lógica de POR QUE o mecanismo resolve o problema.

**Critérios:**
- [ ] Explica a CAUSA RAIZ do problema (Nova Causa)
- [ ] Conecta causa → solução de forma lógica
- [ ] Avatar consegue explicar para outra pessoa

**Exemplo FRACO:**
> "O produto funciona porque tem ingredientes naturais."

**Exemplo FORTE:**
> "O GLP-1 controla a fome no cérebro. Quando está baixo, você sente fome constante. Este ingrediente aumenta GLP-1 naturalmente, por isso você para de sentir fome sem esforço."

---

### M - Mechanism (Mecanismo Único)

**Definição:** O "COMO" específico que diferencia sua solução de todas as outras.

**Critérios:**
- [ ] Tem nome próprio (Gimmick Name)
- [ ] Tem origem crível (Origin Story - lugar, grupo, descoberta)
- [ ] Tem ingrediente/componente hero
- [ ] Pode ser visualizado mentalmente

**Estrutura das 4 Camadas (Puzzle Pieces):**

| Camada | Descrição | Exemplo |
|--------|-----------|---------|
| 1. Ingrediente Hero | Componente principal | Berberina, Sal Rosa |
| 2. Gimmick Name | Nome chiclete | "Pink Salt Trick" |
| 3. Origin Story | De onde veio | "Usado por atrizes de Hollywood" |
| 4. Authority Hook | Gancho de autoridade | "Ozempic Natural" |

---

### B - Believability (Credibilidade)

**Definição:** O quão PROVÁVEL o mecanismo parece para o avatar.

**Critérios:**
- [ ] Passa no teste "Eu contaria isso para um amigo cético?"
- [ ] Tem provas (estudos, depoimentos, autoridades)
- [ ] Não faz claims impossíveis
- [ ] Explica limitações/ressalvas quando necessário

**Níveis de Prova (do mais forte ao mais fraco):**
1. Estudo científico publicado
2. Endosso de autoridade reconhecida
3. Depoimentos verificáveis com nome/foto
4. Lógica interna consistente
5. Analogia com algo conhecido

**Red Flags de Baixa Credibilidade:**
- "Funciona para 100% das pessoas"
- "Resultados em 24 horas"
- "Sem nenhum esforço"
- Números muito redondos/perfeitos

---

### C - Connection (Conexão Emocional)

**Definição:** O mecanismo ressoa com a DRE (Dominant Resident Emotion) do avatar?

**Critérios:**
- [ ] Usa linguagem do avatar (VOC)
- [ ] Toca nas dores/desejos específicos do nicho
- [ ] Cria identificação ("isso sou eu")
- [ ] Gera esperança realista

**Teste de Conexão:**
> Se o avatar ler o mecanismo, ele pensa:
> - "Isso explica por que nada funcionou antes" ✅
> - "Isso parece diferente do que já tentei" ✅
> - "Isso faz sentido para a MINHA situação" ✅

## Checklist de Validação RMBC

```markdown
## Validação RMBC do [NOME DO MECANISMO]

### R - Reason Why
- [ ] Explica causa raiz?
- [ ] Conexão lógica causa → solução?
- [ ] Avatar consegue re-explicar?
Score: __/3

### M - Mechanism
- [ ] Tem Gimmick Name?
- [ ] Tem Origin Story?
- [ ] Tem Ingrediente Hero?
- [ ] Pode ser visualizado?
Score: __/4

### B - Believability
- [ ] Passaria para amigo cético?
- [ ] Tem provas/evidências?
- [ ] Sem claims impossíveis?
- [ ] Tem ressalvas realistas?
Score: __/4

### C - Connection
- [ ] Usa linguagem VOC?
- [ ] Toca DRE do avatar?
- [ ] Gera identificação?
- [ ] Esperança realista?
Score: __/4

**TOTAL: __/15**
- 12-15: APROVADO
- 9-11: Revisar pontos fracos
- <9: Reformular mecanismo
```

## Integração com HELIX

- **Fase 5 (MUP):** Define a Nova Causa que será explicada pelo Reason Why
- **Fase 6 (MUS):** Aplica RMBC completo no mecanismo
- **Fase 8 (Pitch):** Usa Believability para stack de provas
- **Fase 9 (Leads):** Usa Connection para hooks emocionais
